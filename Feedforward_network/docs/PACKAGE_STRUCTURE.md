# Feedforward Package Structure

This repository is organized as a seeded MATLAB live-script release. Active
user-facing scripts are `.mlx` files.

This document describes the `Feedforward_network` package.

## Active folders

- `examples/classification` contains all active seeded classification scripts,
  including MNIST-family image tasks.
- `examples/regression` contains active seeded regression scripts.
- `examples/dynamical_systems` contains the active seeded derivative-field
  dynamical-system trainers, including full-batch, single-seed, and continuation
  wrappers.
- `examples/control` contains the active seeded LQR two-link-arm trainer.
- `examples/pong` contains the active seeded Pong trainer and playback script.
- `tests` contains active seeded tests and seeded weight-consistency checks.
- `src/initializers` contains the deterministic frozen-weight initializers.
- `src/preprocessing` contains reusable time-delay preprocessing helpers.
- `src/dynamical_systems` contains dynamical-system right-hand-side helpers.
- `trained_networks` contains saved seeded `.mat` artifacts.
- `data` contains datasets required by local examples and tests.
- `publication` contains publication plotting and plot-data generation scripts.
- `docs` contains newcomer instructions, dataset preparation, package notes, and
  testing notes.

## Naming policy

Active training scripts use `train_<task>_seeded.mlx` or a clearly named seeded
batch runner such as `train_classification_seeded.mlx`. Active tests use
`test_<task>_seeded.mlx` or a seeded batch runner such as
`test_regression_seeded.mlx`.

Saved seeded artifacts use:

- `trained_networks/seeded/<task>/<task>_seed_<seed>_network.mat` for
  classification, regression, MNIST-family, Pong, and motor control.
- `trained_networks/seeded_state_random_derivative_ode45_lr_0p01/<task>/<task>_seed_<seed>_network.mat`
  for the active derivative-field dynamical-system workflow.
- `trained_networks/seeded_state_random_derivative_ode45_continued_lr_0p001_extra_<epochs>/<task>/<task>_seed_<seed>_network.mat`
  for optional derivative-field continuation training outputs.

The common seed list is `0:9` for all active workflows.

## Leakage policy

Training scripts fit preprocessing statistics from training data only and save
those statistics with the model artifact. Tests require the saved statistics and
split metadata, then evaluate held-out test data only. Older artifacts that lack
this metadata intentionally fail updated tests and should be regenerated with
the matching active seeded training script.

## Reproducibility boundary

All explicit random number generators are seeded. The frozen physical weights
are generated by deterministic custom initializers with the same seed offset for
a given seed across tasks. Exact bitwise equality across machines is still not a
MATLAB guarantee because `trainNetwork`, `dlnetwork`, GPU kernels, CPU math
libraries, and adaptive ODE solvers can differ slightly by platform. Chaotic
closed-loop dynamical systems can amplify those small differences.

## Release Packaging

The clean release package contains active code, documentation, publication plot
scripts, and plot-data generators. It excludes:

- `trained_networks`
- `outputs`
- local dataset `.mat` files in `data`
- generated publication `.mat`, `.mp4`, and `.gif` outputs
- previous release folders
