# Release Notes

## Current release state

The active package has been reduced to the seeded publication workflows. Older
single-network scripts, duplicate MNIST scripts, saved-network tests for old
single-run artifacts, and older dynamical-system NAR/random-window variants are
not part of this public release.

## Dynamical-system workflow

The active dynamical-system workflow is the derivative-field ODE45
implementation. Local DS scripts call a shared trainer and pass fixed seed
lists, task rows, fixed state-space sample seeds, validation settings, Adam
parameters, custom initializer seed offsets, and saved artifact names.

The derivative-field DS trainer uses fixed random normalized state samples,
supervisor `network(x_norm) = x_norm + dx_norm/dt`, validation-loss selection,
and ODE45 testing with `dx_norm/dt = network(x_norm) - x_norm`. The default
training settings are `100000` epochs, validation every `200` epochs, `1000`
training samples, `100` validation samples, training sample seed `12345`,
validation seed `9000`, and reference scaling sample time `1000`. The
publication initial learning rate was `0.01` except for `MO5` seed `3`, `MO13`
seed `3`, `MO7` seed `8`, and `Rikitake` seeds `0` and `4`, which used
learning rate `0.005`. Continuation training is available with learning rate
`0.001`; the publication continuation run used all `MO0` seeds for `50000`
extra epochs and saves to a separate continuation network set.
Local scripts show live plots where applicable.

## Testing policy

Active tests load saved seeded networks and do not retrain. They require saved
split and normalization metadata, use held-out test data, and fail immediately
if required metadata or datasets are missing. Regression tests quote RMSE,
Pearson `r`, and Pearson p-value as mean and standard deviation across seeds.

## Reproducibility policy

All explicit random sources are seeded and the frozen weights are generated by
the deterministic custom initializers. Exact bitwise equality across machines is
not guaranteed because MATLAB training, `dlnetwork` prediction, GPU kernels, CPU
math libraries, and adaptive `ode45` can differ slightly by platform. Long
closed-loop chaotic dynamical-system rollouts can amplify tiny numerical
perturbations, so small cross-machine differences in metrics are expected even
when the code, seeds, weights, and data are the same.
